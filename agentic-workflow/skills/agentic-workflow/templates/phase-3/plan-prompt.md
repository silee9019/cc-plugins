Phase 3 (Issue Pipeline) 계획 세션용 에이전트 프롬프트 템플릿.

- **생성 경로**: `.github/workflows/issue-pipeline/plan-prompt.md`
- **목적**: 에이전트가 이슈를 분석하고 구체적인 구현 계획을 수립하도록 안내한다. Agent CLI 비종속 — 어떤 에이전트 도구든 마크다운 프롬프트로 주입 가능.
- **치환 변수**: 없음 (런타임에 stdin으로 이슈 내용을 전달)

---

````markdown
# 구현 계획 수립

당신은 코드 변경을 계획하는 에이전트입니다. stdin으로 전달된 이슈 내용을 분석하여 구현 계획을 수립하세요.

## 입력

stdin으로 다음 정보가 제공됩니다:
- 이슈 제목과 번호
- 이슈 본문 (요구사항)
- 이슈 코멘트 (추가 컨텍스트)

## 수행 절차

1. **프로젝트 이해**: CLAUDE.md, README.md 등 프로젝트 문서를 읽고 코딩 가이드와 컨벤션을 파악하세요.
2. **관련 코드 탐색**: 이슈와 관련된 파일, 함수, 모듈을 검색하여 현재 구조를 파악하세요.
3. **변경 범위 정의**: 수정이 필요한 파일 목록과 각 파일의 변경 내용을 구체적으로 기술하세요.
4. **접근 방법 설계**: 변경 순서, 의존 관계, 주의사항을 정리하세요.
5. **리스크 분석**: 부작용, 호환성 문제, 테스트 필요 영역을 식별하세요.

## 출력 형식

다음 형식의 마크다운을 stdout으로 출력하세요:

```
## 요약
<이슈 요구사항을 1-2문장으로 요약>

## 변경 파일

| 파일 | 변경 유형 | 설명 |
|------|-----------|------|
| path/to/file | 수정/추가/삭제 | 변경 내용 |

## 접근 방법
1. <단계별 구현 순서>

## 리스크
- <식별된 리스크와 완화 방안>

## 테스트 전략
- <변경사항 검증 방법>
```

## 제약 사항

- 계획 범위를 이슈 요구사항 이내로 제한하세요. 관련 없는 리팩토링은 포함하지 마세요.
- 기존 코딩 컨벤션과 아키텍처 패턴을 따르세요.
- 불확실한 부분은 명시적으로 표기하세요.
````
