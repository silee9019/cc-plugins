# git-init

GitHub 저장소를 초기화합니다: 디렉토리 생성 → GitHub 계정/Author 확인 → `gh repo create` → `.gitignore` → `README.md` → `CLAUDE.md` → `AGENTS.md` 링크 → `.mise.toml` → 초기 커밋 + push.

## Step 0: 발화에서 레포명 추출

**필수**: 레포명. 사용자 발화에서 추출.

| 발화 예시 | → 레포명 |
|---|---|
| "my-project 레포 만들어줘" | `my-project` |
| "awesome-cli 초기화" | `awesome-cli` |
| "GitHub 저장소 만들자 이름은 foo-bar" | `foo-bar` |
| (레포명 명시 없음) | AskUserQuestion으로 확인 |

- kebab-case 검증 (소문자, 숫자, 하이픈만 허용)
- 이미 존재하는 디렉토리인지 확인

## Workflow

### Step 1: 레포명 확인

Step 0에서 추출한 레포명을 `REPO_NAME`으로 확정.
- kebab-case 검증
- 이미 존재하는 디렉토리인지 확인 (있으면 중단)

### Step 2: 프로젝트 정보 수집

발화나 첨부된 계획 파일 등 컨텍스트에서 아래 정보를 최대한 추론합니다.
**추론 가능한 항목은 기본값을 제안하고, 부족한 항목만 AskUserQuestion으로 질문합니다.**

AskUserQuestion 한 번에 아래 항목을 **모두** 묻습니다 (최대 4개 질문):

**기술 스택** (multiSelect, `.gitignore` 및 `.mise.toml` 생성에 사용):

| 옵션 | gitignore.io 키워드 |
|------|-------------------|
| Node.js/TypeScript | `node,react` |
| Kotlin/Gradle | `kotlin,gradle` |
| Rust | `rust` |
| Python | `python` |
| Go | `go` |
| Zsh/Shell | (최소 설정) |

**Visibility**: `public` 또는 `private`

**프로젝트 설명 (영문)**: GitHub description 및 README.md에 사용할 **영문** 한 줄 설명. 한국어 설명은 이를 기반으로 자동 번역하여 `docs/README_ko.md`에 사용합니다.

**Homebrew Tap** (select: `포함` / `미포함`):
- `포함` 선택 시: `.github/workflows/release.yml` + `homebrew-tap` Formula를 자동 생성
- 릴리스 태그 push 시 자동으로 GitHub Release 생성 + homebrew-tap formula 업데이트
- 기본값: CLI 도구 프로젝트면 `포함` 제안, 라이브러리/앱이면 `미포함` 제안

컨텍스트에서 기술 스택이 명확히 판단되면 확인만 받고, 불명확하면 반드시 질문합니다.

### Step 3: 사용자 설정 로드 + GitHub 계정 선택

**캐시 로드**: 플러그인 루트 하위 `data/user-config.json`을 Read로 읽는다. 파일이 없으면 빈 설정(`{}`)으로 시작.

**후보 수집**:
1. `gh auth status` 실행 → 로그인된 GitHub 계정 목록 파싱
2. `git config --global user.name` / `git config --global user.email` 실행

**AskUserQuestion 한 번에 4개 항목 질문** (캐시값이 있으면 기본값으로 표시):
- **GitHub 계정** (select): gh 로그인 계정 목록 제시. 캐시의 `github.account`가 있으면 기본 선택
- **Git repo 기본 폴더 경로** (text): 캐시의 `github.repoBasePath` 또는 빈칸. 레포를 clone할 상위 디렉토리 경로
- **Git Author Name** (text): 캐시의 `git.authorName` > `git config --global user.name`
- **Git Author Email** (text): 캐시의 `git.authorEmail` > `git config --global user.email`

**캐시 저장**: 응답을 `data/user-config.json`에 Write:
```json
{
  "github": {
    "account": "<선택된 계정>",
    "repoBasePath": "<입력된 경로>"
  },
  "git": {
    "authorName": "<입력된 이름>",
    "authorEmail": "<입력된 이메일>"
  }
}
```

**계정 전환**: 선택한 계정이 현재 active가 아니면 `gh auth switch --user <account>` 실행.

### Step 4: 디렉토리 생성 + GitHub 레포 생성

```bash
cd <repoBasePath>/
gh repo create <account>/<repo-name> --<visibility> --clone --description "<프로젝트 설명 영문>"
```

### Step 5: `.gitignore` 생성

**공통 패턴** (모든 프로젝트에 적용):

```gitignore
# === Project-specific (manual) ===

# Claude Code
.claude/settings.local.json

# Environment / Secrets
.env
.env.*
!.env.example

# === Generated by: gi <keywords> ===
# (gitignore.io에서 생성된 내용)
```

**gitignore.io 생성**:

기본 키워드: `macos,windows,linux,intellij+all,eclipse,vim,emacs,sublimetext,visualstudiocode,jetbrains+all`

기술 스택 키워드를 추가하여 아래 URL에서 내용을 가져옵니다:

```
https://www.toptal.com/developers/gitignore/api/<모든키워드쉼표구분>
```

Bash로 `curl`을 사용하여 가져온 후, 공통 패턴 + gitignore.io 내용을 합쳐 `.gitignore`를 생성합니다.

**VSCode 화이트리스트 추가** (Node.js/TypeScript 선택 시):

```gitignore
### VisualStudioCode ###
.vscode/*
!.vscode/settings.json
!.vscode/tasks.json
!.vscode/launch.json
!.vscode/extensions.json
!.vscode/*.code-snippets
```

### Step 6: `README.md` (영문) + `docs/README_ko.md` (한국어) 생성

**이중 언어 README**를 생성합니다. 각 파일 상단에 상대 링크로 다른 언어 버전을 안내합니다.

**`README.md`** (영문, 프로젝트 루트):

```markdown
# <repo-name>

[한국어 (Korean)](docs/README_ko.md)

<프로젝트 설명 영문>

## Tech Stack

- <기술 스택 목록>

## Getting Started

<기술 스택별 시작 가이드 영문>
```

**`docs/README_ko.md`** (한국어):

```markdown
# <repo-name>

[English (영어)](../README.md)

<프로젝트 설명 한국어>

## 기술 스택

- <기술 스택 목록>

## 시작하기

<기술 스택별 시작 가이드 한국어>
```

### Step 7: `CLAUDE.md` 생성

한국어로 작성합니다. **mise task 기반**으로 명령어를 통일합니다:

`CLAUDE.md`는 에이전트가 실제 작업할 때 필요한 운영 규칙만 간결하게 작성합니다.

- 일반적인 프로젝트 구조 트리(`project tree`, `폴더 구조`, `src/`, `tests/` 나열 등)는 넣지 않습니다.
- 구조 설명은 비자명한 아키텍처, 생성 파일, 외부 시스템 연동처럼 작업 판단에 직접 필요한 경우에만 짧게 작성합니다.
- README와 중복되는 사용법/마케팅 설명은 넣지 않습니다.

```markdown
# <repo-name>

<프로젝트 설명>

## 개발

- Language: <언어>
- Task Runner: mise
- Build: `mise run build`
- Test: `mise run test`
- Lint: `mise run lint`
- Format: `mise run fmt`
```

### Step 7.5: `AGENTS.md` 생성

Codex/agent 런타임이 동일한 프로젝트 지침을 읽을 수 있도록 `AGENTS.md`를 `CLAUDE.md`로 향하는 심볼릭 링크로 생성합니다.

```bash
ln -s CLAUDE.md AGENTS.md
```

이미 `AGENTS.md`가 존재하면 덮어쓰지 말고 중단한 뒤 사용자에게 충돌을 알립니다.

### Step 8: `.mise.toml` 생성

기술 스택별 mise task를 정의합니다:

| 기술 스택 | build | test | lint | fmt |
|-----------|-------|------|------|-----|
| Node.js/TypeScript | `pnpm run build` | `pnpm test` | `pnpm run lint` | `pnpm run format` |
| Kotlin/Gradle | `./gradlew build` | `./gradlew test` | `./gradlew ktlintCheck` | `./gradlew ktlintFormat` |
| Rust | `cargo build` | `cargo test` | `cargo clippy -- -D warnings` | `cargo fmt` |
| Python | — | `pytest` | `ruff check` | `ruff format` |
| Go | `go build ./...` | `go test ./...` | `golangci-lint run` | `gofmt -w .` |

추가로 `fmt-check` task도 생성합니다 (CI용 포맷 검증).

### Step 8.5: Homebrew Tap 파이프라인 생성 (선택 시에만)

Step 2에서 **Homebrew Tap: 포함**을 선택한 경우에만 실행합니다.

#### 1. `.github/workflows/release.yml` 생성

기술 스택별로 릴리스 워크플로우를 생성합니다 (Zsh/Shell: 소스 tarball 패턴, Go: GoReleaser, Node/Rust: 크로스 컴파일 바이너리). 상세 템플릿은 원본 `commands/git-init.md`와 동일.

#### 2. `homebrew-tap` 레포에 초기 Formula 생성

`<repoBasePath>/homebrew-tap/Formula/<repo-name>.rb` 경로에 Formula를 생성합니다.
homebrew-tap 디렉토리가 없으면 사용자에게 경로를 확인합니다.

(Zsh/Shell 기본 템플릿과 Go 빌드 템플릿 상세는 원본 명령 문서 참조.)

#### 3. CLAUDE.md 업데이트

CLAUDE.md의 `## 개발` 섹션에 추가:

```markdown
- Release: `git tag vX.Y.Z && git push origin vX.Y.Z`
```

#### 4. Formula 커밋

homebrew-tap 레포에서:

```bash
cd <repoBasePath>/homebrew-tap
git add Formula/<repo-name>.rb
git commit -m "feat: add <repo-name> formula"
git push origin main
```

### Step 9: Git Author 적용 + 초기 커밋 + push

```bash
cd <repo-dir>
git config user.name "<author-name>"
git config user.email "<author-email>"
git add .gitignore README.md docs/README_ko.md CLAUDE.md AGENTS.md .mise.toml
```

Homebrew Tap 포함 시 추가:

```bash
git add .github/workflows/release.yml
```

GoReleaser 패턴 시 추가:

```bash
git add .goreleaser.yaml
```

```bash
git commit -m "chore: initial project setup"
git push -u origin main
```

### Step 10: 후속 안내

완료 후 아래 안내를 출력합니다:

- 레포 URL: `https://github.com/<account>/<repo-name>`
- 로컬 경로: `<repoBasePath>/<repo-name>`
- Git Author: `<author-name> <author-email>`
- AGENTS.md: `CLAUDE.md` 심볼릭 링크 생성 여부

**Homebrew Tap 포함 시 추가 안내:**

- 설치 명령: `brew install <account>/tap/<repo-name>`
- 릴리스 방법: `git tag v0.1.0 && git push origin v0.1.0`
- Formula 경로: `<repoBasePath>/homebrew-tap/Formula/<repo-name>.rb`
- ⚠️ `HOMEBREW_TAP_TOKEN` org secret이 GitHub org에 설정되어 있어야 합니다
- ⚠️ sha256은 `TODO`로 설정됨 — 첫 릴리스 태그 push 시 CI가 자동으로 업데이트합니다

## 주의사항

- `gh auth status`로 GitHub 인증 상태 확인 후 진행
- 디렉토리가 이미 존재하면 중단하고 사용자에게 알림
- gitignore.io 접속 실패 시 공통 패턴만으로 생성
