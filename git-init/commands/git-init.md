# /git-init $ARGUMENTS

GitHub 저장소를 초기화합니다: 디렉토리 생성 → `gh repo create` → `.gitignore` 설정 → 초기 커밋 + push.

## Workflow

### Step 1: 레포명 확인

`$ARGUMENTS`에서 레포명을 추출합니다.

- 레포명이 없으면 AskUserQuestion으로 확인
- kebab-case 검증 (소문자, 숫자, 하이픈만 허용)
- 이미 존재하는 디렉토리인지 확인

### Step 2: 기술 스택 및 visibility 확인

AskUserQuestion으로 아래 항목을 한 번에 확인합니다:

**기술 스택** (multiSelect, `.gitignore` 생성에 사용):

| 옵션 | gitignore.io 키워드 |
|------|-------------------|
| Node.js/TypeScript | `node,react` |
| Kotlin/Gradle | `kotlin,gradle` |
| Rust | `rust` |
| Python | `python` |
| Go | `go` |
| Zsh/Shell | (최소 설정) |

**Visibility**: `public` 또는 `private`

### Step 3: 디렉토리 생성 + GitHub 레포 생성

```bash
cd /Users/silee/ResilioSync/silee-drive/Repositories/silee9019/
gh repo create silee9019/<repo-name> --<visibility> --clone --description "<간단한 설명>"
```

### Step 4: `.gitignore` 생성

**공통 패턴** (모든 프로젝트에 적용):

```gitignore
# === Project-specific (manual) ===

# Claude Code (whitelist)
.claude/*
!.claude/settings.json
!.claude/agents/
!.claude/commands/
!.claude/hooks/
!.claude/skills/

# Environment / Secrets
.env
.env.*
!.env.example

# === Generated by: gi <keywords> ===
# (gitignore.io에서 생성된 내용)
```

**gitignore.io 생성**:

기본 키워드: `macos,windows,linux,intellij+all,eclipse,vim,emacs,sublimetext,visualstudiocode,jetbrains+all`

기술 스택 키워드를 추가하여 아래 URL에서 내용을 가져옵니다:

```
https://www.toptal.com/developers/gitignore/api/<모든키워드쉼표구분>
```

Bash로 `curl`을 사용하여 가져온 후, 공통 패턴 + gitignore.io 내용을 합쳐 `.gitignore`를 생성합니다.

**VSCode 화이트리스트 추가** (Node.js/TypeScript 선택 시):

```gitignore
### VisualStudioCode ###
.vscode/*
!.vscode/settings.json
!.vscode/tasks.json
!.vscode/launch.json
!.vscode/extensions.json
!.vscode/*.code-snippets
```

### Step 5: 초기 커밋 + push

```bash
cd <repo-dir>
git add .gitignore
git commit -m "chore: initial project setup"
git push -u origin main
```

### Step 6: 후속 안내

완료 후 아래 안내를 출력합니다:

- 레포 URL: `https://github.com/silee9019/<repo-name>`
- 로컬 경로: `/Users/silee/ResilioSync/silee-drive/Repositories/silee9019/<repo-name>`
- symlink 필요 시 (zsh 플러그인 등): 명령어 안내

## 주의사항

- `gh auth status`로 GitHub 인증 상태 확인 후 진행
- 디렉토리가 이미 존재하면 중단하고 사용자에게 알림
- gitignore.io 접속 실패 시 공통 패턴만으로 생성
