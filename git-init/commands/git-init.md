# /git-init $ARGUMENTS

GitHub 저장소를 초기화합니다: 디렉토리 생성 → GitHub 계정/Author 확인 → `gh repo create` → `.gitignore` → `README.md` → `CLAUDE.md` → `.mise.toml` → 초기 커밋 + push.

## Workflow

### Step 1: 레포명 확인

`$ARGUMENTS`에서 레포명을 추출합니다.

- 레포명이 없으면 AskUserQuestion으로 확인
- kebab-case 검증 (소문자, 숫자, 하이픈만 허용)
- 이미 존재하는 디렉토리인지 확인

### Step 2: 기술 스택, visibility, 프로젝트 설명 확인

AskUserQuestion으로 아래 항목을 확인합니다:

**기술 스택** (multiSelect, `.gitignore` 및 `.mise.toml` 생성에 사용):

| 옵션 | gitignore.io 키워드 |
|------|-------------------|
| Node.js/TypeScript | `node,react` |
| Kotlin/Gradle | `kotlin,gradle` |
| Rust | `rust` |
| Python | `python` |
| Go | `go` |
| Zsh/Shell | (최소 설정) |

**Visibility**: `public` 또는 `private`

**프로젝트 설명**: "이 프로젝트를 한 문장으로 설명해주세요" (README.md 및 GitHub description에 사용)

AskUserQuestion으로 git author 정보를 확인합니다:

- **Git Author Email**: 기본값 `silee9019@gmail.com`

이 값은 Step 9에서 `git config user.name` / `git config user.email`로 적용됩니다.

### Step 4: 디렉토리 생성 + GitHub 레포 생성

```bash
cd /Users/silee/ResilioSync/silee-drive/Repositories/silee9019/
gh repo create silee9019/<repo-name> --<visibility> --clone --description "<프로젝트 설명>"
```

### Step 5: `.gitignore` 생성

**공통 패턴** (모든 프로젝트에 적용):

```gitignore

# Claude Code (whitelist)
.claude/*
!.claude/settings.json
!.claude/agents/
!.claude/commands/
!.claude/hooks/
!.claude/skills/

# Environment / Secrets
.env
.env.*
!.env.example

# === Generated by: gi <keywords> ===
# (gitignore.io에서 생성된 내용)
```

**gitignore.io 생성**:

기본 키워드: `macos,windows,linux,intellij+all,eclipse,vim,emacs,sublimetext,visualstudiocode,jetbrains+all`

기술 스택 키워드를 추가하여 아래 URL에서 내용을 가져옵니다:

```
https://www.toptal.com/developers/gitignore/api/<모든키워드쉼표구분>
```

Bash로 `curl`을 사용하여 가져온 후, 공통 패턴 + gitignore.io 내용을 합쳐 `.gitignore`를 생성합니다.

### Step 6: `README.md` 생성

한국어로 작성합니다. 아래 템플릿을 기반으로 프로젝트명, 설명, 기술 스택을 반영합니다:

```markdown

<프로젝트 설명>

## 기술 스택

- <기술 스택 목록>

## 시작하기

<기술 스택별 시작 가이드>
```

### Step 7: `CLAUDE.md` 생성

한국어로 작성합니다. **mise task 기반**으로 명령어를 통일합니다:

```markdown
# <repo-name>

<프로젝트 설명>

## 프로젝트 구조

(초기 단계 — 구조 확정 후 업데이트)

## 개발

- Language: <언어>
- Task Runner: mise
- Build: `mise run build`
- Test: `mise run test`
- Lint: `mise run lint`
- Format: `mise run fmt`
```

### Step 8: `.mise.toml` 생성

기술 스택별 mise task를 정의합니다. 내부 명령어 참조:

| 기술 스택 | build | test | lint | fmt |
|-----------|-------|------|------|-----|
| Node.js/TypeScript | `pnpm run build` | `pnpm test` | `pnpm run lint` | `pnpm run format` |
| Kotlin/Gradle | `./gradlew build` | `./gradlew test` | `./gradlew ktlintCheck` | `./gradlew ktlintFormat` |
| Rust | `cargo build` | `cargo test` | `cargo clippy -- -D warnings` | `cargo fmt` |
| Python | — | `pytest` | `ruff check` | `ruff format` |
| Go | `go build ./...` | `go test ./...` | `golangci-lint run` | `gofmt -w .` |

추가로 `fmt-check` task도 생성합니다 (CI용 포맷 검증).

### Step 9: Git Author 적용 + 초기 커밋 + push

```bash
cd <repo-dir>
git config user.name "<author-name>"
git config user.email "<author-email>"
git add .gitignore README.md CLAUDE.md .mise.toml
git commit -m "chore: initial project setup"
git push -u origin main
```
